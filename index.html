02 Supabase: создание таблиц

Зайди в Supabase → SQL Editor и выполни этот скрипт целиком:

SQL → Supabase SQL Editor
-- 1. Расширение для векторов (RAG)
create extension if not exists vector;

-- 2. Пользователи
create table users (
  id          bigserial primary key,
  tg_id       bigint unique not null,
  tg_username text,
  plan        text default 'free',
  checks_used int  default 0,
  created_at  timestamptz default now()
);

-- 3. Технические задания
create table tz_docs (
  id         bigserial primary key,
  user_id    bigint references users(id),
  title      text,
  content    text,
  created_at timestamptz default now()
);

-- 4. Чанки ТЗ с эмбеддингами (RAG)
create table tz_chunks (
  id        bigserial primary key,
  tz_id     bigint references tz_docs(id) on delete cascade,
  user_id   bigint references users(id),
  chunk_idx int,
  content   text,
  embedding vector(1024) -- Jina v3 / DeepSeek размер
);

-- 5. Результаты проверок
create table checks (
  id          bigserial primary key,
  user_id     bigint references users(id),
  tz_id       bigint references tz_docs(id),
  post_text   text,
  result_json jsonb,
  score       int,
  created_at  timestamptz default now()
);

-- 6. Аналитика событий воронки
create table analytics_events (
  id         bigserial primary key,
  user_id    bigint,
  event_name text,
  properties jsonb,
  created_at timestamptz default now()
);

-- Функция векторного поиска
create or replace function match_tz_chunks (
  query_embedding vector(1024),
  match_user_id   bigint,
  match_count     int default 5
)
returns table(id bigint, content text, similarity float)
language sql
as $$
  select id, content,
    1 - (embedding <=> query_embedding) as similarity
  from tz_chunks
  where user_id = match_user_id
  order by embedding <=> query_embedding
  limit match_count;
$$;
Важно: В Supabase → Authentication → Policies включи Row Level Security (RLS) для таблиц users, tz_docs, checks — иначе данные доступны всем.

03 Воркфлоу 1 — Приём и роутинг сообщений

Это главный диспетчер. Все сообщения из Telegram приходят сюда, здесь определяется тип и запускается нужная ветка.

📱
1 Telegram Trigger Trigger
Стартовая нода. Получает все входящие апдейты от бота.
Тип нода: Telegram Trigger
Updates to Watch: message
Credential: Telegram API (введи Bot Token от @BotFather)
👤
2 Upsert User HTTP Request
Регистрирует или обновляет пользователя в Supabase при каждом обращении.
Method: POST
URL: https://[project].supabase.co/rest/v1/users
Header: Prefer: resolution=merge-duplicates
Body (JSON):
{"tg_id": "{{$json.message.from.id}}", "tg_username": "{{$json.message.from.username}}"}
🔀
3 Message Router Switch
Определяет тип сообщения и направляет в нужную ветку.
Нода: Switch

Ветка 0 — /start или /help
Условие: {{$json.message.text}} starts with /start

Ветка 1 — Загрузка ТЗ
Условие: {{$json.message.text}} starts with /upload_tz

Ветка 2 — Проверка поста
Условие: {{$json.message.text}} starts with /check

Ветка 3 — Голосовое сообщение
Условие: {{$json.message.voice}} exists

Ветка 4 — Документ (файл ТЗ)
Условие: {{$json.message.document}} exists
👋
4 Send Welcome Telegram
Ответ на /start. Объясняет как работать с ботом.
Нода: Telegram → Send Message
Chat ID: {{$json.message.chat.id}}
Text: Привет! Я ТЗ-Чекер 🤖\n\n📄 /upload_tz — загрузить ТЗ (текстом или файлом)\n✅ /check [текст поста] — проверить пост\n📊 /stats — моя статистика
Parse Mode: Markdown
🔗
5 Call Sub-Workflows Execute Workflow
Для веток 1–4 вызываются соответствующие под-воркфлоу через ноду Execute Workflow.
Нода: Execute Workflow
Ветка /upload_tz → Воркфлоу 2 (RAG Upload)
Ветка /check → Воркфлоу 3 (Check Post)
Ветка voice → Воркфлоу 4 (Voice STT)
Режим: Wait for sub-workflow

04 Воркфлоу 2 — Загрузка ТЗ (RAG Indexing)

Пользователь присылает текст ТЗ или документ. Воркфлоу нарезает его на чанки, создаёт эмбеддинги и сохраняет в Supabase pgvector.

📨
1 Workflow Input Execute Workflow Trigger
Принимает данные от главного воркфлоу.
Нода: Execute Workflow Trigger
Входные данные: tg_user_id, message_text, file_id (если документ)
📁
2 Get File (если документ) HTTP Request
Скачивает файл из Telegram если пришёл документ вместо текста.
Нода: HTTP Request
URL: https://api.telegram.org/bot[TOKEN]/getFile?file_id={{$json.file_id}}
Далее: GET https://api.telegram.org/file/bot[TOKEN]/{{file_path}}

Нода: Extract From File (тип: текст или PDF)
✂️
3 Chunk Text Code
Нарезает ТЗ на чанки по 500 символов с перекрытием 100 символов для лучшего поиска.
Нода: Code (JavaScript)

const text = $input.first().json.tz_text; const size = 500, overlap = 100; const chunks = []; for (let i = 0; i < text.length; i += size - overlap) { chunks.push({ content: text.slice(i, i + size), idx: chunks.length }); } return chunks.map(c => ({json: c}));
🔢
4 Create Embeddings HTTP Request
Для каждого чанка создаёт векторное представление через Jina AI (бесплатно 1M токенов/мес).
Нода: HTTP Request
Method: POST
URL: https://api.jina.ai/v1/embeddings
Header Authorization: Bearer [JINA_API_KEY]
Body:
{"model": "jina-embeddings-v3", "input": ["{{$json.content}}"], "task": "retrieval.passage"}
💾
5 Save TZ Doc + Chunks HTTP Request × 2
Сначала создаёт запись ТЗ, затем в цикле сохраняет все чанки с эмбеддингами.
Нода 5a: HTTP Request → POST /rest/v1/tz_docs
Body: {"user_id": ..., "title": "ТЗ от [дата]", "content": "..."}
Header: Prefer: return=representation (чтобы получить новый id)

Нода 5b: HTTP Request → POST /rest/v1/tz_chunks
Body: {"tz_id": ..., "user_id": ..., "chunk_idx": ..., "content": ..., "embedding": [...]}
✅
6 Confirm to User Telegram
Уведомляет пользователя об успешной загрузке.
Text: ✅ ТЗ загружено! Сохранено {{$json.chunk_count}} фрагментов.\nТеперь отправь /check [текст поста] для проверки.
Альтернатива Jina: Можно использовать модель text-embedding-3-small от OpenAI (очень дёшево — $0.02/млн) или локальный Ollama с моделью nomic-embed-text если у тебя есть VPS.

05 Воркфлоу 3 — Проверка поста (RAG + DeepSeek)

Ключевой воркфлоу. Получает текст поста, ищет релевантные части ТЗ через векторный поиск и отправляет в DeepSeek для анализа.

📝
1 Parse Post Text Set
Извлекает текст поста из команды /check.
Нода: Set
post_text: {{$json.message.text.replace('/check', '').trim()}}
user_tg_id: {{$json.message.from.id}}
🔢
2 Embed Query HTTP Request
Создаёт эмбеддинг для текста поста (для векторного поиска).
URL: https://api.jina.ai/v1/embeddings
Body task: "retrieval.query" (!) — другой task чем при индексации
Input: ["{{$json.post_text}}"]
🔍
3 Vector Search в Supabase HTTP Request
Вызывает RPC-функцию match_tz_chunks для поиска релевантных частей ТЗ.
Method: POST
URL: https://[project].supabase.co/rest/v1/rpc/match_tz_chunks
Body:
{"query_embedding": {{$json.embedding}}, "match_user_id": {{$json.db_user_id}}, "match_count": 5}
🔧
4 Build Prompt Code
Собирает финальный промпт из найденных чанков ТЗ и текста поста.
Нода: Code (JavaScript)

const chunks = $input.all().map(i => i.json.content).join('\n---\n'); const post = $('Parse Post Text').first().json.post_text; const prompt = `Ты — строгий проверщик постов блогеров. ТЕХНИЧЕСКОЕ ЗАДАНИЕ (извлечённые фрагменты): ${chunks} ТЕКСТ ПОСТА ДЛЯ ПРОВЕРКИ: ${post} Проверь пост по каждому пункту ТЗ. Для каждого критерия выведи: ✅ ВЫПОЛНЕН — [критерий] — [краткое объяснение] ❌ НЕ ВЫПОЛНЕН — [критерий] — [что именно отсутствует] В конце: ИТОГ: X/N критериев выполнено. Отвечай только на русском языке.`; return [{json: {prompt}}];
🧠
5 DeepSeek API Call HTTP Request
Отправляет промпт в DeepSeek и получает анализ поста.
Method: POST
URL: https://api.deepseek.com/v1/chat/completions
Header Authorization: Bearer [DEEPSEEK_API_KEY]
Body:
{"model": "deepseek-chat", "messages": [ {"role": "system", "content": "Ты эксперт по проверке контента."}, {"role": "user", "content": "{{$json.prompt}}"} ], "max_tokens": 2000, "temperature": 0.1}
📊
6 Parse Score Code
Извлекает числовой счёт из ответа DeepSeek для сохранения в БД.
const text = $json.choices[0].message.content; const match = text.match(/(\d+)\/(\d+)/); const score = match ? parseInt(match[1]) : 0; return [{json: {result_text: text, score}}];
💾
7 Save Check Result HTTP Request
Сохраняет результат проверки в таблицу checks.
Method: POST
URL: https://[project].supabase.co/rest/v1/checks
Body:
{"user_id": ..., "tz_id": ..., "post_text": "{{$json.post_text}}", "result_json": {"text": "{{$json.result_text}}"}, "score": {{$json.score}}}
📤
8 Send Result to Telegram Telegram
Отправляет отформатированный результат пользователю.
Chat ID: {{$('Workflow Input').first().json.chat_id}}
Text: {{$json.result_text}}
Parse Mode: Markdown

06 Воркфлоу 4 — Голосовой ввод (Groq Whisper)

Обрабатывает голосовые сообщения: скачивает аудио из Telegram, транскрибирует через Groq (бесплатно) и передаёт текст в Воркфлоу 3.

📥
1 Get Voice File Info HTTP Request
Получает путь к голосовому файлу через Telegram Bot API.
URL: https://api.telegram.org/bot[TOKEN]/getFile
Params: file_id = {{$json.message.voice.file_id}}
⬇️
2 Download Voice File HTTP Request
Скачивает .ogg файл с серверов Telegram как бинарные данные.
URL: https://api.telegram.org/file/bot[TOKEN]/{{$json.result.file_path}}
Response: Binary (File)
🎙
3 Groq Whisper STT HTTP Request
Транскрибирует аудио через Groq API (бесплатно 28 800 сек/сутки).
Method: POST
URL: https://api.groq.com/openai/v1/audio/transcriptions
Header Authorization: Bearer [GROQ_API_KEY]
Body type: multipart/form-data
Fields:
file: [binary из предыдущей ноды] model: whisper-large-v3-turbo language: ru response_format: json
🔗
4 Pass to Check Workflow Execute Workflow
Передаёт расшифрованный текст в Воркфлоу 3 как обычный пост.
Нода: Execute Workflow
Workflow: Воркфлоу 3
Input: post_text = {{$json.text}} (из ответа Groq)
Тип: Голосовое → подставляем как /check
Регистрация Groq: Зайди на console.groq.com → API Keys → Create Key. Бесплатный tier включает Whisper и LLM модели (Llama 3, Mixtral). Можно использовать Groq и вместо DeepSeek для ещё большей скорости.

07 Воркфлоу 5 — Дашборд и аналитика

Webhook-эндпоинт для загрузки данных в UI на GitHub Pages. Пользователь открывает страницу — браузер вызывает этот вебхук, получает статистику.

🌐
1 Webhook Trigger Webhook
Принимает GET запросы от браузера с Telegram user ID для авторизации.
Нода: Webhook
HTTP Method: GET
Path: /dashboard
Authentication: None (защита через tg_init_data — см. ниже)
Response: Last Node
🔐
2 Validate Telegram Auth Code
Проверяет подпись Telegram WebApp initData для безопасной авторизации.
Нода: Code (JavaScript)
Логика: HMAC-SHA256 проверка строки initData с Bot Token как ключом. Если подпись валидна — извлекаем user.id

// Псевдокод проверки: const crypto = require('crypto'); const secret = crypto.createHmac('sha256', 'WebAppData') .update(BOT_TOKEN).digest(); const hash = crypto.createHmac('sha256', secret) .update(dataCheckString).digest('hex');
📊
3 Fetch Stats from Supabase HTTP Request × 3
Параллельно загружает статистику: общий счёт, историю проверок, список ТЗ.
3a: GET /rest/v1/checks?user_id=eq.{{uid}}&select=*&order=created_at.desc&limit=20
3b: GET /rest/v1/tz_docs?user_id=eq.{{uid}}&select=id,title,created_at
3c: GET /rest/v1/checks?user_id=eq.{{uid}}&select=score&order=created_at.desc&limit=100
📦
4 Format Response Code
Собирает данные в единый JSON для отправки в браузер.
return [{json: { checks: $('Fetch Checks').all().map(i => i.json), tz_list: $('Fetch TZ').all().map(i => i.json), avg_score: /* среднее из scores */, total_checks: checks.length }}];
📤
5 Respond to Webhook Respond to Webhook
Отправляет JSON-ответ браузеру с CORS заголовками.
Нода: Respond to Webhook
Response Body: {{$json}}
Response Code: 200
Headers: Access-Control-Allow-Origin: *

08 Деплой: GitHub Pages + n8n

Настройка n8n

Самый простой путь — n8n.cloud (бесплатный план до 5 воркфлоу и 2 500 выполнений/мес).

  1. Зайди на n8n.cloud → Sign Up → создай аккаунт
  2. Создай каждый из 5 воркфлоу как описано выше
  3. Для Telegram: Settings → Telegram API Credential → вставь Bot Token
  4. Для Supabase: в каждой HTTP Request ноде добавь заголовки apikey и Authorization: Bearer [SUPABASE_ANON_KEY]
  5. Активируй все воркфлоу (кнопка Active в правом верхнем углу)

Настройка Telegram бота

  1. Напиши @BotFather → /newbot → получи токен
  2. Активируй вебхук: вставь в браузер URL:
    https://api.telegram.org/bot[TOKEN]/setWebhook?url=https://[n8n-host]/webhook/[webhook-path]
  3. Команды бота → /setcommands → установи: start - Начать, upload_tz - Загрузить ТЗ, check - Проверить пост, stats - Статистика

GitHub Pages (лендинг + UI)

  1. Создай репозиторий на GitHub (например tzchecker)
  2. Загрузи landing.html как index.html и ui.html как dashboard.html
  3. В dashboard.html замени URL вебхука дашборда на твой n8n URL:
    fetch('https://[n8n-host]/webhook/dashboard?initData=...')
  4. Settings → Pages → Branch: main → Save. Сайт появится на [username].github.io/tzchecker
Self-hosted n8n: Если хочешь неограниченное количество выполнений — установи n8n на любой VPS (Digital Ocean $4/мес, Oracle Cloud бесплатно). Команда: docker run -d -p 5678:5678 n8nio/n8n. Затем открой ngrok или настрой nginx для публичного URL.

09 Авторизация через Telegram WebApp

Чтобы пользователь dashboard.html видел только свои данные — используем Telegram Login Widget или Telegram WebApp initData.

Вариант A: Telegram WebApp (проще)

В dashboard.html добавить в <head>:

<script src="https://telegram.org/js/telegram-web-app.js"></script>
<script>
  // Telegram заполняет window.Telegram.WebApp.initData автоматически
  const initData = window.Telegram.WebApp.initData;
  const user = window.Telegram.WebApp.initDataUnsafe.user;

  // Передаём initData в n8n для верификации
  fetch(`https://[n8n]/webhook/dashboard?initData=${encodeURIComponent(initData)}`)
    .then(r => r.json())
    .then(data => renderDashboard(data));
</script>

Вариант B: Telegram Login Widget

Добавь на лендинге кнопку входа через Telegram. Пользователь кликает → Telegram верифицирует → на твой URL приходят данные пользователя с подписью.

<script async src="https://telegram.org/js/telegram-widget.js"
  data-telegram-login="[BOT_USERNAME]"
  data-size="large"
  data-auth-url="https://[n8n-host]/webhook/tg-auth"
  data-request-access="write">
</script>
